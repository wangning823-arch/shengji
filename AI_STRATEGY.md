# 拖拉机AI出牌策略总结

> 基于 `advanced-ai.js` 代码分析，核心原则：**让对手得最少的分**

---

## 一、牌力评估系统

## 三、首家出牌策略 (`selectLeadPlay`)

### 3.1 对手缺门感知

通过 `CardTracker.isPlayerVoidInSuit(seat, suit)` 检测对手缺门情况：
- 两个对手都缺门：该花色评分 -30（非常危险，容易被将吃）
- 一个对手缺门：该花色评分 -15（较危险）
- 无对手缺门：评分不变（安全）

### 3.2 手牌数量因子

- 手牌 ≤ 4张：评分 +5（后期更激进，用大牌抢分）
- 手牌 ≥ 20张：评分 -5（前期更保守，保存实力）
- 其他：评分不变

### 3.3 出牌优先级

**核心原则：不能确保最大的情况下，不出分牌(K=10分, 10=10分, 5=5分)**

| 优先级 | 策略 | 评分 | 说明 |
|--------|------|------|------|
| 0.5 | 副牌K(同花色有A对) | 125 | A对保底，先出K跑10分，对手可能只有一张该花色 |
| 1 | 副牌A对子 | 120 | 确定最大的牌，先出 |
| 2 | 副牌A单张 | 110 | 确定最大的牌 |
| 2.5 | 分对(对手全缺门+主对已清) | 115 | 对手无法跟牌且无主对将吃，安全跑分 |
| 3 | 副牌K对子(确保最大) | 105 | 有A保护，或A已出过 |
| 3.5 | 主牌对子(有分对需保护) | 100 | 有分对在对手缺门花色时，先清主对保护分对 |
| 4 | 副牌K单张(确保最大) | 100 | 有A保护，或A已出过 |
| 5 | 副牌拖拉机 | 95 | 长套连续对子 |
| 6 | 长套副牌甩牌 | 80 | 4+张且最大是A/K |
| 7 | 主牌对子/拖拉机 | 80 | 对子比单张清主更高效 |
| 8 | 其他副牌对子 | 75 | 非分牌对子(Q,J等) |
| 9 | 小主牌清主 | 70 | 清主比出裸副牌更重要 |
| 10 | 小王 | 70 | 保底概率低，积极清主 |
| 11 | 副牌非分牌(Q,J,9,8,7,6,4,3) | 55 | 无战略意义的裸牌，优先级低于清主 |
| 12 | 副牌K对子(无A保) | 50 | 不安全，等有A再出 |
| 13 | 副牌K单张(无A保) | 45 | 不安全，等有A再出 |
| 14 | 副牌10/5(有A保) | 55 | 有A保护时可出 |
| 15 | 副牌10/5(无A保) | 35 | 不安全，等有A再出 |
| 15.5 | 分对(对手全缺门+主对未清) | 60 | 等主对清完再出，避免被将吃 |
| 16 | 大王 | 20 | 最低优先级，保留保底 |

**分牌保护机制**：
- K：需要同花色有A（手中或已出过）才安全出，否则评分45
- 10：需要同花色有A才安全出，否则评分35
- 5：需要同花色有A才安全出，否则评分35
- A已出过后，K/10/5变为该花色最大牌，可以安全出

**非分牌出牌顺序**：从大到小出(Q>J>9>8>7>6>4>3)，更可能赢得出牌权。

**清主优先于裸副牌**：没有A/K/对子/拖拉机时，优先出小主牌/小王清主（70分），而不是出无意义的裸副牌7等（55分）。清主可以削弱对手将吃能力。

**队友无主限制**：当队友确认无主时（通过`isPlayerVoidInSuit(seat, 'trump')`检测），不出小于K的主牌。原因：队友无主意味着1v2，出小主牌对手可以轻松管住并用K/10跑分。只出K及以上的大主牌清主。

**分对保护战术**：当有副牌分对(10/5对)且对手全部缺门该花色时：
- 对手无法跟牌，但可能用主对将吃
- 策略：先出主对（评分100）消耗对手主对，再出分对（评分115）安全跑分
- 判断条件：通过已出主对数(≥4)判断对手主对是否已耗尽
- 例外：自己没有主对可清时，直接出分对（无法主动清主）

## 四、跟牌策略 (`selectFollowPlay`)

### 4.1 四种情况分支

```
队友出牌 + 队友在赢 → selectTeammateWinningPlay
队友出牌 + 对手在赢 → selectTeammateLosingPlay
对手出牌 + 队友在赢 → selectOpponentLeadingButTeammateWinningPlay
对手出牌 + 对手在赢 → selectOpponentWinningPlay
```

### 4.2 情况1：队友出牌 + 队友在赢 (`selectTeammateWinningPlay`)

**末家（队友确定最大）**：
- 贴同花色分牌 (95)
- 没同花色时：垫副分(90) > 垫最小副牌(70)
- 有同花色时：跟最小同花色(65)
- **不浪费主牌将吃**（队友已赢，主分将吃无意义）

**非末家**：
- 跟主牌：最小非分主牌(82) > 最小分主牌(65)
- 跟副牌：最小非分牌(82) > 最小分牌(65)
- 没同花色：将吃(85+对手分×3) > 垫分牌(80) > 垫最小牌(75)

**原则**：队友在赢时出最小牌，不浪费大牌；非末家避免出分牌（防对手管住得分）。

### 4.3 情况2：队友出牌 + 对手在赢 (`selectTeammateLosingPlay`)

- 有同花色：
  - **能赢 + 非末家 + 有好副牌**（A/拖拉机/大对子）：出**小王/级牌/A**抢出牌权（排除大王留保底，排除K/10/5不浪费分牌）
  - **能赢 + 非末家 + 无好副牌**：出**最小能赢的牌**（如A够用就不出大王/小王/级牌），保留大牌保底
  - **不能赢 + 非末家**：优先不出分牌（10/K），出最小非分牌
- 跟不了：出最小牌(40)
- 没同花色：将吃(80+分×3) > 垫最小牌(50)

### 4.4 情况3：对手出牌 + 队友在赢 (`selectOpponentLeadingButTeammateWinningPlay`)

**末家（队友确定最大）**：
- 同情况1的末家策略（不浪费主牌将吃）

**非末家**：
- 跟主牌：最小非分主牌(82) > 最小分主牌(65)
- 跟副牌：最小非分牌(75) > 最小分牌(60)
- 没同花色：将吃(80+对手分×3) > 垫副分(78) > 垫最小副牌(70)

**原则**：不要破坏队友优势，出最小牌；有分牌时将吃接管。

### 4.5 情况4：对手出牌 + 对手在赢 (`selectOpponentWinningPlay`)

- 有同花色：
  - **能赢 + 非末家 + 有好副牌**（A/拖拉机/大对子）：出**小王/级牌/A**抢出牌权（排除大王留保底，排除K/10/5不浪费分牌）
  - **能赢 + 非末家 + 无好副牌**：出**最小能赢的牌**（如A够用就不出大王/小王/级牌），保留大牌保底
  - **不能赢 + 非末家**：优先不出分牌（10/K），出最小非分牌；只有分牌时出最小分牌
  - **末家**：出最小能赢的牌（节省实力）
- 跟不了：出最小牌(30+牌值)
- 没同花色：将吃(80+分×3) > 垫最小牌(40)

## 五、将吃(Chop)策略

### 5.1 将吃候选生成 (`generateChopCandidates`)

**非末家**：
- trick有分牌：出最大主牌将吃（保护分数）
- trick无分牌：出最小主牌将吃（节省实力）

**末家 + 场上有人将吃**：
- 队友将吃了 → 不再管，垫牌
- 对手将吃了 → 尝试超将吃（出更大的主牌）
- 超不了 → 垫最小非分副牌(60) > 最小非分主牌(55) > 最小主牌(50)

**末家 + 无人将吃**：
- 优先用主分牌将吃（+5分奖励）
- 没主分 → 最小主牌

### 5.2 牌型匹配

将吃时严格匹配首家牌型：
- 单张 → 单主将吃
- 对子 → 对主将吃
- 拖拉机 → 拖拉机主将吃
- 找不到匹配牌型时退回垫牌

### 5.3 将吃评分

```
将吃评分 = baseScore + trickPoints × 3
```

场上分牌越多，将吃越优先。例如对手贴30分时，将吃评分 = 80 + 90 = 170。

## 六、扣底策略 (`selectBottomCards`)

| 优先级 | 操作 | 说明 |
|--------|------|------|
| 高 | 扣非主分牌(10/K/5) | 分牌危险，避免被对手抠底 |
| 中高 | 扣主牌分牌(主5/主10/主K) | 被抠底时翻倍扣分，比非分主牌更危险 |
| 中 | 扣非主小牌 | 保留大牌 |
| 低 | 扣主牌非分牌 | 主牌尽量保留 |

**保护机制**：不拆对子（对子优先级-20）

## 八、保底机制

- 所有策略函数最终通过 `validatePlay` 验证
- 验证失败时进入 `generateFallbackCandidates` 备选
- 最终保底：出 `hand[0]`

## 八.5、底牌保护策略（保底意识）

### 8.5.1 核心概念

- 底牌分数（`bottomCardPoints`）：庄家扣底时放入的分牌总值
- 保护等级（`bottomProtection`）：0=无需保护，1=中等保护，2=强保护
- 最后一轮赢家获得底牌分数；闲家抠底时分数翻倍（单张×2，对子×4，拖拉机×6）

### 8.5.2 保护等级判定

**庄家方**（可见底牌）：
| 条件 | 保护等级 |
|------|----------|
| 后期(≤4轮) + 底分≥30 | 2（强保护） |
| 后期(≤4轮) + 底分≥15 | 1（中保护） |
| 底分≥40（前期大底） | 1（中保护） |

**闲家方**（根据已出牌估算底分）：
| 条件 | 保护等级 |
|------|----------|
| 后期(≤4轮) + 估算底分≥25 | 2（强保护） |
| 后期(≤4轮) + 估算底分≥10 | 1（中保护） |

### 8.5.3 对出牌策略的影响

**庄家方（保护底牌）**：
- 大王评分：20 → 75（强保护）/ 45（中保护）→ 积极出牌抢赢
- 小王评分：50 → 70（强保护）/ 60（中保护）→ 积极出牌抢赢

**闲家方（抢底）**：
- 大王评分：保持20 → 保留用于最后一轮抢底
- 小王评分：50 → 35（强保护）→ 保留用于最后一轮抢底
- 小主牌清主评分：50 → 35（强保护）→ 保留大牌

**跟牌策略**：
- 闲家方有强保底需求时，跟主牌始终出最小能赢的牌（保留大牌用于最后一轮）

## 九、核心设计原则

1. **最小化对手得分**：所有决策以此为最终目标
2. **非末家不出分牌**：防对手管住得分
3. **末家安全送分**：队友确定最大时贴分牌，不浪费主牌将吃
4. **K保护**：无A保护时降级K的价值
5. **大王保底**：大王评分极低(20)，尽量不出（底分多时例外）
6. **将吃优先级随分牌提升**：trickPoints越多，将吃越重要
7. **将吃根据分牌决定大小**：有分牌出最大保护，无分牌出最小节省
8. **对手缺门感知**：出牌前检查对手缺门情况，避免被将吃
9. **手牌数量因子**：后期更激进，前期更保守
10. **底牌保护意识**：庄家方底分多时用大牌保底，闲家方估算底分多时保留大牌抢底
11. **非末家防末家跑分**：对手在赢时，有好副牌则出小王/级牌/A抢出牌权（大王留保底，K/10/5不浪费）；无好副牌则出最小能赢的牌
12. **分对保护战术**：有副牌分对(10/5)且对手全部缺门时，先出主对消耗对手主对，再安全出分对跑分
13. **队友无主不出小主**：队友确认无主时，不出小于K的主牌（1v2局面，对手可轻松管住跑分），只出K及以上大主牌清主

## 十、已修复问题记录

### ✅ 问题1：非末家跟主牌时"出最大能赢的牌"过于激进
- **修复**：trick有分牌时出最大保护，无分牌时出最小节省
- **影响**：减少大牌浪费

### ✅ 问题2：非末家将吃时"出最大主牌"过于激进
- **修复**：trick有分牌时出最大主牌，无分牌时出最小主牌
- **影响**：减少大王/小王浪费

### ✅ 问题3：末家"主分将吃"在队友确定最大时浪费主牌
- **修复**：删除末家队友确定最大时的主分将吃选项
- **影响**：保留主牌用于后续管住对手

### ✅ 问题4：没有利用CardTracker的缺门信息
- **修复**：首家出牌时通过`isPlayerVoidInSuit`检测对手缺门，降低缺门花色的出牌评分
- **影响**：避免出对手缺门的花色被将吃

### ✅ 问题6：没有考虑手牌数量对策略的影响
- **修复**：添加手牌数量因子，后期(+5)更激进，前期(-5)更保守
- **影响**：策略更灵活

### ✅ 问题7：副牌出牌顺序不合理
- **修复**：
  - A单张评分110（最大的牌，优先出）
  - K/10/5分牌需要A保护才安全出，否则评分低(35-45)
  - A已出过后，K/10/5变为最大牌，可以安全出
  - 非分牌(Q,J,9,8,7,6,4,3)从大到小出，安全无分牌损失
- **影响**：确保不会在没有保护的情况下送出分牌

### ✅ 问题10：扣底策略对主牌分牌处理粗糙
- **修复**：主牌分牌优先级-20（比非分主牌-50更容易被扣掉）
- **影响**：减少被抠底时的分数损失

### ✅ 问题11：跟牌策略中"最小分牌(无奈)"评分过高
- **修复**：评分从75/68降到65/60
- **影响**：确保有非分牌可选时不会误选分牌

### ✅ 问题13：AI没有保底意识
- **修复**：
  - `analyzeGameState` 新增 `bottomCardPoints`（底牌实际分值）和 `bottomProtection`（保护等级0/1/2）
  - 庄家方：底分≥30(后期)时大王评分20→75，小王评分50→70
  - 闲家方：估算底分≥25(后期)时小王评分50→35，小主牌评分50→35（保留大牌抢底）
  - 跟牌：闲家方有强保底需求时，始终出最小能赢的主牌
- **影响**：庄家方底分多时用大牌保底，闲家方底分多时保留大牌抢底

### ✅ 问题14：K保护逻辑不严谨（2副牌场景）
- **修复**：原逻辑只检查"是否有A已出过"，改为检查"该花色所有A是否全部已知（手中+已出 = deckCount）"
- **原代码**：`playedSuitCards.some(c => c.rank === 'A')` → 只要有1个A出过就认为K安全
- **新代码**：`(acesInHand + acesPlayed) >= deckCount` → 必须所有A都已知才认为K安全
- **影响**：2副牌场景下，不会在只出过1个A时就认为K最大，避免送出分牌

### ✅ 问题15：跟主牌时出分牌（10/K）送给对手
- **场景**：对手出主牌3，队友跟6，AI（非末家）出10 → 10是分牌，对手会管住得分
- **修复**：
  - `selectOpponentWinningPlay` 和 `selectTeammateLosingPlay` 的 losing plays 部分：跟主牌非末家时，优先出非分牌（3/4/6/7/8/9/J/Q），避免出10/K送给对手
  - 只有当手牌全是分牌时，才出最小分牌
- **影响**：避免在跟主牌时白送分牌给对手

### ✅ 问题16：跟主牌时出小牌，让末家跑分
- **场景**：对手出主牌3，队友跟6，AI出3 → 末家可以用K赢（10分），AI应该出A抢出牌权
- **修复**：`selectOpponentWinningPlay` 和 `selectTeammateLosingPlay` 的 winning plays 部分：对手在赢时，非末家出**最小能赢的牌**（如A够用就不出大王/小王/级牌），既防末家跑分又保留大牌保底
- **影响**：对手在赢时用A等大牌抢出牌权，但不浪费大王/小王/级牌

### ✅ 问题17：分对(10/5)在对手缺门时被将吃
- **场景**：AI有副牌10对，对手全部缺门该花色，但对手还有主对 → 直接出10对被对手主对将吃，损失20分
- **修复**：
  - `selectLeadPlay` 新增分对保护战术：检测分对所在花色对手是否全部缺门
  - 对手可能还有主对时：主对评分提升到100（先清主对），分对评分降低到60（等待）
  - 对手主对已耗尽（已出主对≥4）或自己无主对可清时：分对评分115（安全出牌）
  - 通过 `cardTracker.playedBySuit` 统计已出主对数判断对手主对剩余
- **影响**：避免在对手有主对时送出分对，先清主对再安全跑分

### ✅ 问题18：队友无主时出小主牌送分
- **场景**：队友已无主牌，AI出小主牌清主 → 对手轻松管住，用K/10跑10分
- **修复**：
  - `llm-ai.js` CardTracker：新增主牌缺门检测，当lead是主牌且某玩家跟不了时标记`voidMap[seat]['trump']`
  - `selectLeadPlay`：检测队友是否无主（`isPlayerVoidInSuit(teammate, 'trump')`），队友无主时只出K及以上的大主牌清主，不出小主牌
- **影响**：避免在1v2局面下出小主牌送分给对手

## 十一、未修复问题（低优先级）

### 问题5：甩牌策略可能不合法
- **原因**：需要复杂的合法性验证，`validatePlay`会拒绝不合法的甩牌
- **影响**：中等，被拒绝后会走fallback

### 问题8：将吃风险评估不足
- **原因**：需要复杂的概率计算，评估下家是否有更大的主牌
- **影响**：中等，当前策略已经根据分牌价值调整

### 问题9：`ensureCorrectPlayCount` 可能改变策略意图
- **原因**：结构性问题，需要重构策略函数生成正确数量的候选
- **影响**：低，补齐逻辑优先补同花色牌

### 问题12：没有"拆对手对子"的策略
- **原因**：需要推断对手手牌，非常复杂
- **影响**：低，属于高级战术
